# Mossven — agent instructions

## Always: changelog first

1. **Before doing any work in this repo, read [`CHANGELOG.md`](./CHANGELOG.md).** It is the
   running log of what has happened, what changed, and which files were touched.
2. **After making any change, append an entry to `CHANGELOG.md`** following the format
   described at the top of that file (date, what/why, `Touched:` files, any non-obvious notes).

This keeps every session and every agent in sync. Do not skip it.

## Always: frontend-design skill

For **any** frontend or UI work in this repo — building or styling pages, components,
demos (`/ukazky/*`), landing sections, artifacts — read and apply
[`SKILL.md`](./SKILL.md) (the `frontend-design` skill) first. It defines the bar for
distinctive, production-grade design and the aesthetics to follow (and the generic
"AI slop" patterns to avoid). Treat it as the design standard for this project, not
an optional reference.
